# Приложение для отслеживания движений человека

## Билд

Установить в python окружение все зависимости (выполнить в папке [python/motion](./src/python/motion/)): ```pip install -r requirements.txt```

Выполнить билд приложения в корне репозитория: ```dotnet build build/build.proj -t:CleanAll,BuildAll```. Эта команда очистит все bin/obj файлы и gRPC контракты в python части и снова соберет dotnet часть и python контракты

Просмотреть файл [mediapipe_models/readme](./src/python/motion/mediapipe_models/README.md)

## Запуск
1. python/motion часть запускается с команды ``` python -m app.main```
2. dotnet часть запускается либо из .exe в bin у [dotnet/MotionCapture/](./src/dotnet/MotionCapture/). Либо другим удобным способом, например, через Visual Studio

## Обновления
### 10.04.26
1. Добавлена калибровка камеры с сохранением в файл. Вместе с тем изменено API, добавлен метод калибровки, работает в режиме стриминга
2. Сделана возможность вкл/выкл захват и калибровку. По умолчанию всё выключено, включение одного выключает второе
3. Большой рефакторинг работы с кадром от момента захвата до отправки на питон сервер. Изменения:
    * Теперь VM не работает с сервисами захвата движения и калибровки, между ними добавлена прослойка
    * Сервис синхронизации кадров взял на себя применение FPS к захвату. Ивент в нем вызывает прослойку к сервисам захвата и калибровки
    * Добавлен стейт приложения для источника истины по вопросам вкл/выкл различных функций
    * Точки на скелете окрашиваются в зависимости от стороны (лево/право/центр) и видимости (точки с низкой видимостью обновляются реже и окрашены в красный). В процессе калибровки точки внутренних углов шахматной доски выводятся на кадр с % калибровки 
4. Изменена фильтрация и сглаживание точек
5. Добавлен HealthCheck, после любого неудачного запроса приложение блокируется до следующего удачного. После разблокирования надо заново запустить захват движений/калибровку. Вместе с этим Connection State Service с которым работает UI и интерсептор gRPC
6. Добавлен вывод ошибок запросов на UI (Overlay Control)
7. В API разделен MotionService для лучшего соответствия логике (появился Camera Service)
8. Добавление gRPC клиента в .Net части перешло в рефлексию, в конфиге можно создать клиента, который будет соответствовать .proto файлу. Можно настроить адрес и политику повторных запросов.
9. Перенос VM на CommunityToolkit.Mvvm

### 09.03.26

3. Повышена скорость работы (получение списка камер и открытие камеры). Требуется рефакторинг сервисов камеры
